# 推送到 GitHub Pages 的步驟

目標：把這個資料夾推到 https://github.com/zway-education/K12-calendar
完成後：https://zway-education.github.io/K12-calendar/ 上線

---

## 一、先確認 GitHub repo 存在

進 https://github.com/zway-education
如果**沒有** K12-calendar 這個 repo：
1. 點右上 **+ → New repository**
2. Repository name：`K12-calendar`
3. **Public** (要公開才能用 Pages 免費)
4. **不要勾** Add README / .gitignore / license (我們已經有了)
5. **Create repository**

---

## 二、終端機指令（複製貼上一次跑）

打開 PowerShell，cd 到這個 deploy 資料夾，然後跑：

```powershell
cd "C:\Users\KHUser\Desktop\Claude\●K12教學營運\K12_收費行事曆\_deploy_K12-calendar"

git init
git branch -M main
git add .
git commit -m "Initial deploy: K12 收費行事曆 v1"
git remote add origin https://github.com/zway-education/K12-calendar.git
git push -u origin main
```

> 第一次推可能會要求 GitHub 登入（瀏覽器彈出授權）

---

## 三、啟用 GitHub Pages

1. 進 https://github.com/zway-education/K12-calendar
2. **Settings** → 左側選單 **Pages**
3. Source 選 **Deploy from a branch**
4. Branch：**main** / Folder：**/ (root)**
5. **Save**
6. 等 1-2 分鐘，刷新會看到 ✅ Your site is live at https://zway-education.github.io/K12-calendar/

---

## 四、之後更新流程

```powershell
cd "C:\Users\KHUser\Desktop\Claude\●K12教學營運\K12_收費行事曆\_deploy_K12-calendar"

# 把最新的原型同步進 deploy 資料夾 (Claude 改完原型後手動跑)

cp ../_雲端設定包/* ./setup/

git add .

git push
```

GitHub Pages 約 30 秒~1 分鐘自動部署最新版。

## 五、整合到「招生發文規劃_主檔」

主檔的 iframe src 改成線上 URL 就不用煩惱相對路徑：

**現在**（相對路徑，本機用）：
```html
<iframe src="../../●K12教學營運/K12_收費行事曆/K12_收費行事曆原型_v1.html" ...>
```

**改成**（絕對 URL，線上用）：
```html
<iframe src="https://zway-education.github.io/K12-calendar/" ...>
```

要我幫您改這一行的話告訴我「主檔 iframe 改成線上 URL」。

## 注意事項

- **個資保護**：`.gitignore` 已排除 `_基準資料/`(內含學生姓名/電話)。**不會** push 到 public repo
- **Apps Script 部署**：與 GitHub Pages 無關。Sheet + Apps Script 仍照 [setup/README_設定步驟.md](setup/README_設定步驟.md) 在您的 K12 教育帳號上設定
- **雲端網址**：使用者第一次打開 https://zway-education.github.io/K12-calendar/ 時,需在「⑤ 規則與設定」貼上 Apps Script 網址,瀏覽器會記在本機

## 疑難排解

| 症狀 | 處理 |
|---|---|
| `git push` 跳「Permission denied」 | 您的 GitHub 帳號沒有 zway-education org 的 push 權限。請去 org 設定加自己,或請有權限的人代 push |
| `git push` 跳「remote: Repository not found」 | repo 還沒建,回到「一、先確認 GitHub repo 存在」 |
| 推完了但 Pages 沒上線 | 確認在 Settings → Pages 選了 main / root |
| 中文資料夾 push 後 GitHub 顯示亂碼 | Git 預設不支援中文,但檔案內容沒問題,只是檔名顯示。GitHub Pages 仍可正常運作 |
